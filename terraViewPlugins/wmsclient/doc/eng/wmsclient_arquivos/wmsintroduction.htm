Introduction to WMS Support in TerraView

<< Previous | First  | Next >>

Web Map Service, or WMS, is a type of service to produce maps in the Web from geo-referred information. TerraView version 3.2.0 provides support to create maps (themes) from WMS servers. Starting from this version, TerraView can be a WMS services client, requesting and presenting maps from these type of service.

The user can connect to servers compliant with official versions WMS specifications defined by the Open Geospatial Consortium (OGC). These WMS server versions are:

  • 1.3.0 from 03/15/2006, 03-109r1;

  • 1.1.1 from 01/16/2002, 01-068r3;

  • 1.1.0 from 01/21/2001, 01-047r2;

  • 1.0.0 from 04/19/2000, 00-028.

The WMS Client plugin provides TerraView with the following capabilities:

  • Creation of themes WMS servers to be visualized with other common TerraView themes. There is no limit to the number of servers and maps that can be added to the same view;

  • Display of legend of a WMS theme when the server provides correctly this support;

  • Query of information about features displayed on maps from WMS servers, through pointing on the draw area (maps visualization);

  • Query of information about WMS themes: server address, metadata provided by the server, information about projection and map presentation.

Figure below shows TerraView main window. Observe in the figure that there are 3 themes associated to Brasil view:

  • Municípios (Municipalities): map of Brazilian municipalities from the database to which TerraView is currently connected;

  • Biomas do Brasil (Brazil Biomes): map from a WMS, highlighted with an different Theme icon - T letter added by W symbol – to indicate that the theme is from a layer originated from a WMS server;

  • Limites estaduais do Brasil (Brazil States Borders): map from a WMS.

TerraView window displaying maps from WMS servers

In the window shown above, two WMS themes are from the same WMS. However, this is not a restriction given that the user can add in a view maps from different services. To execute that, open the WMS creation window to add a new map in TerraView window. Click here to learn how to create themes from WMS services or continue to the next page.

More Details about WMS Services :

The above functionalities are possible since a WMS server provides a set of standard operations, which are:

  • GetCapabilities: returns a XML document describing the interfaces are supported by the server, the layers that are served, and the image formats used to serve the layers;

  • GetMap: allows clients to request “figures” that correspond ti served maps;

  • GetFeatureInfo: provides additional information to clients about specific points in the image returned by the server.
