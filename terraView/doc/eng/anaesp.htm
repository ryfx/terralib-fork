Spatial Statistics

Global Moran Index

This index is used to identify the spatial correlation structure that best describes the data. The basic principle is to characterize the spatial dependency by showing how values are spatially correlated. In a general way, Moran index is useful to test if the null hypothesis is that there is spatial dependency; in this case the value is zero. Positive values (between 0 and +1) indicate a direct correlation and negative values (between 0 and -1) indicate inverse correlation. Once the index is calculate, it is important to statistically validate it.

Activate UP Theme with BH.mdb open. In the menu bar, select the Analysis - Spatial Statistics. The Spatial Statistics window will be displayed, as shown in the figure below.

1. Define the Attribute for which the presence of local spatial correlation must be tested. Select the attribute homicides in year 2003 (HOM_03).

2. Activate the Moran Index option.

3. To evaluate the significance of the index, select 99 permutations.

4. Click on OK to execute the procedure.

5. Test result will be shown in the Index Value and p-value fields.

It can be verified that there is no spatial autocorrelation in the Homicides in year 2003 since the Moran index value is 0.105837, with 0.12 significance. A small index value and a significance value over 0.05, indicates that the hypothesis of no spatial autocorrelation can not be rejected.
A visual analysis can also be executed using a Thematic Map. For instance, select five equal steps for the focus attribute in the Edit Legend option on the UP Theme popup menu.

If there were spatial autocorrelation, it can be expected that, areas with greater (smaller) proportions of homicides were closer to each other. However, the map above shows that this does not happen, since no clear pattern can be observed.

To practice with an example where there is spatial autocorrelation, follow the steps above using the Mean Income (rendamedia) attribute.
After the procedure is applied, it can be verified that there is spatial autocorrelation with the Mean Income attribute, since the Moran index value is 0.446818, with 0.01 significance value.

Local Moran Index - LISA

Global autocorrelation indicators, such as Moran index, provide one value as the measure of spatial association for the whole study region dataset. However, in many cases it is desirable to analyze patterns in a scale with more details to verify if the hypothesis of stationarity of the process is valid locally.
To verify the hypothesis, it is necessary to use spatial association indicators that can be associated to different location of a spatially distributed variable. This methodology uses the Local Moran Index to find spatial correlation in these areas. Given that it is a local indicator, there is a specific correlation value for each area.

Activate UP Theme with the BH.mdb database open. In the menu bar, select Analysis - Spatial Statistics. A new window will be displayed, as shown in the following figure.

1. Define the Attribute for which the presence of local spatial correlation must be tested. Select rendamedia (mean income).

2. Activate the Local Moran Index (LISA) option.

3. To test the significance of the index, select 99 permutations.

4. Insert the Prefix of the column to be generated. Type in RENDA.

5. Click on OK to execute the procedure.

The following columns are created:

- Z: Vector with observed deviations.

- Wz: Local weighted average vector.

- MoranIndex: Local Moran Index.

- LISASig: Value of the p statistics. In the given example, if the value is greater than 0.05 we may assume that there is no autocorrelation and it is significant.

- BoxMap: Presented values correspond to the relation between Z and Wz values in a Dispersion Plot divided by quadrants (Q). Values range from 1 to 4, where 1 corresponds to values in Q1 (high-high – high Z and Wz values), 2 (Q2, low-low), 3 (Q3, high-low), and 4 (Q4, low-high). In the study case, if the deviations vector (Z) and the averages vectors (Wz) are high – case of the Q1 quadrant - it may be assumed that it is an UP with high average income surrounded by other UPs in the same situation; and if it is Q2, the UP has low average income and is surrounded by UPs in the same situation.
Cases Q3 and Q4 detect Ups with high and low incomes located near UPs with lower and higher incomes, respectively. Note that this type of study only divides UPs in two groups in relation to average income (high and low).

- LISAMap: This column values, ranging from 0 to 4, are created only if a level of significance is selected in the interface.
During the creation of a LISAMap, local indices are classified as: non significant (0) or significant with confidence 95% (1, p=0.05), 99% (2, p=0.01) and 99,9% (3, p=0.001).

- MoranMap: This column values are generated only if a level of significance is selected in the interface. This result presents only regions where Ii values are significant (interval above 95%) while BoxMap does not consider significance. In this case, local indices Ii are associated to the Moran dispersion diagram.
The values are: 0 (non significant), 1 – Q1 (high-high), 2 – Q2 (low-low), 3 – Q3 (high-low), 4 – Q4 (low-high)..

Local Mean

Spatial Local Mean method examines the mean value mi of an attribute in the study region (first order).

Activate UP Theme with the BH.mdb database open. In the menu bar, select Analysis - Spatial Statistics. A new window will be displayed, as shown in the following figure:

1. Define the Attribute for which the presence of local spatial correlation must be tested. Select the rendamedia attribute.

2. Activate the Local Mean option.

3. Evaluate the test significance by selecting 99 permutations.

4. Insert the Prefix of the column to be generated. Type in RendaMedia.

Create a legend using Quantil mode to produce a Thematic Map with the generated column.

G and G* Functions

G and G* functions are two local spatial autocorrelation indices that allow the test of hypotheses about the spatial clustering of the values sum, associated to the neighbors points to the considered one.
Since this indicators are composed by the sum of attributes values, the observation of significant high values of Gi and Gi* indicates that this attributes occur in high values, and the opposite indicates clustering of low values.
The main difference between G and G* functions is that the former considers values of all neighbors while the later considers the region to calculate the index.

Activate UP Theme with BH.mdb database open. In the menu bar, select Analysis - Spatial Statistics. A new window will be displayed, as shown in the following figure:

1. Define the Attribute for which the presence of local spatial correlation must be tested. Select the rendamedia attribute.

2. Activate the G option.

3. To test the significance of the index, select 99 permutations.

4. Define a Prefix of the column to be generated by the G function (this is an optional step).

5. Click on OK to execute the procedure.

A new column will be created with the values of the G function. With this estimation, a Thematic Map can be created.
Follow the steps presented above, but using the G* option.
