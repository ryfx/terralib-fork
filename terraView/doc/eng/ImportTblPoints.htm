Importing Point Data

 

Geographic information about points may be available in table format. The geographic position of each points is defined by X and Y coordinates, and, for instance, may be gathered by GPS. It is required that each line of the table corresponds to a single point.

Tables of data points can be in DBF or text formats. In DBF format, the names and types of the columns are defined by the file which contains the table, on the other hand, in text format, they must be informed during the import. In the example below, a table of points is in text format, also known as CSV, which stands for "Comma Separated Values".

To import this kind of table, choose the File > Import Table of Points menu option. Fill in the necessary parameters in the displayed Import Table of Points window.

Follow these steps:

  1. Select the file with the table of points. The available formats are DBF and CSV. Click on File and select the fire_spot_sp.csv file, available from TerraView tutorial (do not forget to change the file selection filter to show only those with “.csv” extension).

  2. Choose the separator character (of column values in each line) from the available ones in the interface, or another of your choice. In this case, Comma is the character.

  3. Inform if the file first line is a header with the columns name. In this case, select Yes.

  4. For each file column, choose its type, name, maximum number of characters (if the type is text), and format (if the type is date/time). Click on Apply button.

  5. Choose the name of the layer to be created.

  6. Inform the projection of the coordinates, if it is known. in this case, they are in LatLong projection with Datum WGS84.

  7. Define in the Geometry Information field the columns corresponding to points coordinates. In this case, the Lon column contains the X coordinate and the Latitude column contains the Y coordinate. They are defined of Real type because they are in decimal degrees. If they were in degrees, minutes and seconds, as are columns LatGMS and LonGMS in the same figure, they should be defined of Text type, in order to be correctly imported and be able to be used as points coordinates. Values of both columns must use the "." (dot) character as the decimal point separator between the integer and fraction parts.

  8. Inform if any of the columns is used as the unique identifier of each object represented by the point. It will be used as the link between the attributes table and the objects (geometries). If there is no such column, use the Automatic option. In this example, use the Nr column.

  9. Click on OK button to execute.
