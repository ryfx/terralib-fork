Renaming and Deleting Layers, Views and Themes

When right clicking on a Layer, View, and Theme, the corresponding popup menu is activated, as seen in the example below for a View:

  • Rename: allows the user to change the name of a layer, view, or theme, to a more appropriate one.
  • Remove: deletes from the Database a Layer, a View, or a Theme.
  • Properties: provides the properties of a layer, view, or theme.

If a Layer is removed from Databases Tree using the Remove option, information is deleted permanently from the database. In this case, all its themes will be deleted. When a view is deleted, all its themes will also be deleted. In this case, the themes may be rebuilt since a new theme can be created using data from the source Layer.

Create a new View called Crimes and add Crimes2000, Crimes 2001, Crimes2002, and Crimes2003 themes. Move all themes of Crimes View to BH_Novo View. Delete the empty Crimes View from the Views Tree.

Delete BH_Novo View to revert to the original database.
