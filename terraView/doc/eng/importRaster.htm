Raster Import

 

A raster is composed by a set of points, called "pixels", organized as rows and columns. A scanner captures an image and assigns a row, a column, and a color value (black and white, gray level, or color) to each point. Satellite images and aerial photographs are examples of raster data.
TerraView imports the following file types: TIF, JPG, RAW, and TXT. To import, execute these steps:

1. Click on the menu File - Raster Import as shown in the Figure below.


Access to raster import through the Menu

2. The Raster Import interface will be displayed as shown in the Figure below.


Initial Raster Import interface

3. Use File button to indicate where the image is stored and click on Open button.

4. Once the raster image to be imported is selected, the number of Rows and Columns will be presented.


Interface after choosing the image input file

Dummy parameter is optional and, if defined, indicates the pixel value considered to be transparent, that is, no information is attached to it. During import, the pixel value will be mapped to the output Dummy value, the one considered transparent in the image stored in the database. Pixels with this value will not be drawn in the Draw Area.

5. Click on Next button to move to the next window, as shown in the Figure below.


Geographical Characteristics interface

6. This window presents information extracted from the input raster file, and also after having changed the projection, as explained below.

- The projection information may sometimes not be the one of the image in the input file. If possible, get the correct projection information from the image provider.

7. If the presented projection is not the correct one, change it by clicking on Projection, providing the correct parameters of the projection and clicking on OK to return to the Geographical Characteristics interface.

8. When the projection is correct, click on Next button. The interface in the Figure below will be shown.


Storage Characteristics interface

9. Fill in other items as presented in the sequence:

- Name: if a new layer is going to be created, its name must be typed in. If the layer already exists, select it from the presented list of layers. The last case occurs when a mosaic is beeing create using a second file;

- Object Id: currently this field must not be filled in;

- Compression: to be applied before data is stored to reduce the physical space and can be: none, zlib, or jpeg. Jpeg compression is the most efficient, although it can not be applied to pallete type images;

- Width and Height define the size of data block used in storage; recommended value is 512 x 512;

- Dummy: defines the transparent value, as mentioned before. This value is suggested by the import interface using the same value defined for the input image, presented in the interface shown in step 4, and detailed in the text that follows the figure;

- Expansible: must be used when various files create one image, that is, when a mosaic is being created and this is the first imported file.

10. Click on Next, to proceed to the next step regarding multiresolution, or click on Finish button to start the import.

11. The optional multiresolution step intends to enhance image visualization performance, by using a pyramid with several images obtained from the original image using different resolutions, the higher the resolution the closer to the bottom of the it. The highest resolution corresponds to the original image in the input file, which will be always imported at the bottom of the pyramid (level 0).

The following parameters about pyramid must be set:

- Number of (additional resolution) Levels to be used in the pyramid. This value depends on the application and visualization scale usually used. If the scale is much smaller than the image original one, this value must be high (for instance, 10);

- Select the additional levels in the list at right: initially the list is shown with the provided number of levels already selected, which are consecutive and starting from level 1. This choice is recommended, but it is possible to select any levels, even if they are not consecutive and in a quantity which is different from the number provided before.
