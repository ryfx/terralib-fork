Importing Geographic Data


1. Once a TerraView database is created, geographic data composed by maps and tables can be imported into the database. Select  icon in the toolbar to import geographic data. The import operation is accessed also by File > Import Data menu and is executed through the Import interface that will be displayed.

2. Choose File option to access the file with data to import. A directory browser window will be displayed.

3. Choose the file to import. Available data formats that can be imported are: ArcView (Shape files); MapInfo (MID files); and SPRING (SPRING-Geo files). Go to Data directory, select Shape file (*.shp) type, and choose Bairros.shp file. Click Open button.

4. Once the file has been selected, TerraView will go back to the Import interface in order to define the cartographic projection of the data. Click Projection.

5. Projection window is used to define geographic data Projection, Datum, and their parameters. Select UTM (Universal Transverse Mercator), SAD69, and Zone 23. Click OK.

6. Select the column to link attributes to geometries. This column can be created automatically by TerraView. Here choose the Automatic option.

7. Click OK and the data will be imported to the database in a new layer. Before that, the interface asks you to confirm the previous choices.

Imported data can be automatically visualized in TerraView. This option is selected in the interface:

If the selection is No, the layer is created and inserted in the Layers Tree but is not displayed. In this case, select Yes. TerraView creates a new "VistaBairros" View and a new "Tema_Bairros" Theme. With a View and a Theme, the imported data is displayed in the Draw Area. Otherwise, when display option is No, a View is created, but the Theme is not shown. Adding a Theme requires clicking on Theme button and requesting the Theme to be inserted in the View.
