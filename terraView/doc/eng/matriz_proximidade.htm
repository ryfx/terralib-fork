Spatial Proximity Matrix

A basic tool to estimate the data spatial variability in an area is the Spatial Proximity Matrix, also known as Neighborhood Matrix. The proximity matrix is used to calculate indicators during the exploratory analysis of spatial data for other analysis available in TerraView. Since the calculation is costly and can be executed with criteria, TerraView stores the matrices in the database, allowing the use of them in different analysis to avoid recalculation, thus achieving efficiency.

To calculate the proximity matrix, select the Analysis - Proximity Matrix - Create Proximity Matrix option in the Menu Bar. The interface shown in the following figure below will be displayed. Once the operation is finished, the created matrix describes, for each object, which other objects are considered to be the nearest ones.

1. Choose the Theme with data. Choose UP Theme with the Planning Units for Belo Horizonte in the BH.mdb database.

2. Select one of the available strategies in Construction Strategy field to decide how objects are close to each other. In this case, select the Contiguity strategy. The options are:

a. Contiguity: An object is near to only other ones with shared borders. This characteristic is highly dependent on objects geometry and the Theme objects geometries must be without digitalization and building errors;

b. Distance: An object is near another one if their centroids are at a distance d specified by the user in the Distance field (enabled when this strategy is selected);

c. Nearest Neighbours: An object is near on only the n nearest neighbors (considering centroids), where n is defined by the user in Number of Neighbours field (enabled when this strategy is selected).

3. Weight field allows weights to be assigned to the found proximities. The options are:

a. Without Weight: All near objects have the same weight;

b. Inverse Distance;

c. Squared Inverse Distance.

4. Save output file option allows the matrix calculated by TerraView to be saved in text files for data interchange with other spatial analysis software. In this case, fill in the format and the location and name of the file.

5. Click on OK.

The matrix is built and saved in the database, associated with this Theme.

Selecting a Proximity Matrix

Using Analysis Proximity Matrix - Select Proximity Matrix menu, the interface shown in Figure below is displayed and it allows the manipulation of the proximity matrices built for a given Theme.

1. Choose the Theme with proximity matrices.

2. Observe the list of available matrices.

3. Click on Set Matrix as Default to indicate that the selected matrix is the one to be used in the following spatial operations which need a proximity matrix.

4. Click on Delete Matrix to delete the selected matrix.
