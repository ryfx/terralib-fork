Creation of Distance Zones or Buffers

A buffer or distance zone is an area around an object with a fixed distance from it. Some examples of buffer are shown in Figure 1.


Figure 1 - Examples of distance zone or buffer

The interface to create buffers or distance zones is shown in Figure 2. This interface creates a new layer with all buffers created from the objects of an input theme and a given distance. To activate the interface, use the context menu on a View, and Geoprocessing – Create Buffers option.


Figure 2 - Create buffers interface

The interface has 6 parts:

1. Input: used to define the input theme and which theme objects are considered (all, pointed only, or queried only.

2. Distance: used to define a fixed distance for all objects or to select the numeric attribute which defines specific distances for each object.

IMPORTANT: the distance unit MUST be the same of the data. For instance, if data unit is decimal degrees, the distance must be also in decimal degrees; if the unit is meters, the distance must be in meters, and so on. To check which is the input theme unit, verify the base layer projection of the theme.

3. Multiple Levels: different buffer levels based on a distance d can be created. Therefore, each level n buffer corresponds to a distance buffer between d*(n-1) and d*n. In Figure 3, 3 buffer levels were created around Minas Gerais state borders, considering a 7000 meters distance. The first level is a 7000 meters buffer. The second level corresponds to a distance buffer between 7000 and 14000 meters. And finally, the third level corresponds to a distance buffer between 14000 and 21000 meters.


Figure 3 - Multiple levels

4. Dissolve Barriers Between Buffers: each computed buffer for each object of an input theme creates a new object in the output layer. Depending on the selected distance, the output layermay contain overlapping objects. If dissolve barriers between barriers option is selected, all overlapping objects in the output layer are joined in a single object. This operation is shown in Figure 4.


Figure 4 - Dissolve barriers between buffers

The input theme is shown in Figure 4.a. The output layer created without dissolving barriers between buffers is shown in Figure 4.b. Observe that each buffer created one object in the output layer (buffer_1, buffer_2 and buffer_3) and that they overlap. The output layer created by dissolving barriers between buffers is shown in Figure 4.c. In this case, the output layer is created with one object only (buffer_1), resulting from the union of all created buffers.

5. Rule To Polygon Case: this part of the interface is enabled when the geometric representation of the input layer is polygonal. Three types of buffer can be selected: Only Inside, Inside and Outside, and Only Outside. Figure 5 shows each type of buffer. Figure 5.a shows a buffer created with only the inside region of Minas Gerais state, Figure 5.b shows a buffer created with only the outside region, and Figure 5.c shows a buffer created with inside and outside regions.


Figure 5 - Buffer types

6. Output: used to define the output layer name. the created layer will contain the buffers and and a static table with distance and level of each buffer.
