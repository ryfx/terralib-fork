Importing External Tables


When a table with data to import has the linking column different from the id of a layer of geographic objects (Primary Key), this data cannot be associated to the layer. The Table can be imported as an External Table and later be associated with a theme. Import procedure is the same as seen previously, differing only by selecting the link type to be External. The Table is imported without being associated to any layer or theme. Import txdesemprego Table. Then link to UP theme with the following steps.

To link this External Table to a theme, activate the popup menu for the theme and select Link External Table option.

The Linking External Table interface is displayed and data of all attribute Tables available in the system are presented.

1. Select the Table to link to the theme (txdesemprego).

2. Select the linking column (object_id_) by clicking on its name. A linking column must allow the identification of the object presented in the External Table (a table line) as the same geographic object presented in the layer. In addition, the column must contain unique ids of objects.

3. Drag the cursor, keeping the mouse button pressed down, to the Grid column with matching information. Cursor will be displayed with a not allowed symbol until it is on a Grid column name, then the symbol will change to a drop symbol. Drag the mouse to the column COD_UP label in the Grid and release the button to apply the linkage.

4. Observe that new information about unemployment rate is available as part of the attributes Table associated with UP theme.

The difference between table data is that if you import a Table to a layer, this layer is available to create a theme. Otherwise, importing as an external table requires the Table to be linked to the theme after being created.
