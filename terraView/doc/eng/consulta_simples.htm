Simple Query

Open BH.mdb database. Observe that this database contains the layers AreasVerdes, Bairros, Batalhões, Colégios, CompanhiasPM, Crimes2000, Crimes2001, Crimes2002, Crimes2003, Favelas, Lagoa, Limite, PostosCombustivel, Rios, RuasCentrais, SetoresCensitarios, and UP. Activate Bairros theme and check in the Grid area the attributes associated to this geographic object. The column named "POP2000" represents the population in the year 2000 in each neighborhood. By using an attribute query, neighborhoods with population over 6000 inhabitants can be selected.

1. Activate Bairros theme and visualize only it;

2. Use the popup menu;

3. Select Attribute Query option.

Attribute Query interface will be displayed.

1. Define the attribute to be used in the query. Select POP2000.

2. Operators field defines the type of operation to execute on the attribute. Select > . Available operators in TerraView are:

  Operator   Meaning
=   Equal to value  
<>   Different from value  
>   Greater than value  
>=   Greater than or equal to value  
<   Smaller than value  
<=   Smaller than or equal to value  
LIKE   String comparison  
IS NULL      No value for attribute (empty entry)  
 IS NOT NULL     There is value for attribute (non empty entry)  

3. In Attribute Values field, define the value to be used in the comparison. By selecting this option, all column values are shown in the list on the right. The value can also be typed in the Edit field. Select this button and choose 6000.

4. Click New Query.

Observe that some lines are now highlighted in the Grid area. These lines correspond to query results: neighborhoods with population over 6000 inhabitants are selected. To visualize on the map, activate Bairros theme and use Reset button. A window similar to the one shown below will be displayed. Observe that the neighborhoods are highlighted with query color.

Some comparison operators

In addition to usual comparison operators, such as =, >, <, <>, <=, >=, TerraView makes available three other useful operators : LIKE, IS NULL and IS NOT NULL.

The purpose of each one of these operators and examples using them are presented in the following items.

LIKE

LIKE comparison operator can be used when all values of a column has to be queried for coincident characters.

Use the popup menu of Crimes2003 theme and select Attribute Query option. Define attribute as NAT, the operator as LIKE, and click on Edit field. Type in 'C%' after the operator. Click New Query button.

The query will show all values of NAT column that start with the C letter. In this case, NAT column presents crime codes that happened in Belo Horizonte during the year 2003, and the codes starting with letter C refer to Robbery. Check in the Grid area that all robbery types are highlighted. The operator entry (in this case 'C%') indicates how to build a query. This entry must be between apostrophe (' ') or double quotes (" "); Percentage symbol (%) indicates that any set of characters (including absence of one) can occupy the position. Therefore, in the example, all values starting with C letter are selected. For the case where values to query must end with character 7, the command would be typed NAT LIKE '%7'; if selected values must contain character 2 at any position, the command would be typed NAT LIKE '%2%'; and if the last selection must not contain values ending with character 2, the command would be typed NAT NOT LIKE '%2' and Filter button would be clicked.

IS NULL

IS NULL comparison operator can be used to query values of a given column for absent values. Use the popup menu of Colegios theme and select Attribute Query option. Define the attribute as ENSINOMEDI and the operator as IS NULL. This operator does not require an entry and nothing has to be added to the command. Click New Query button.

Schools with high school (“Ensino Médio”) are highlighted with an 'X' and the others have no value in ENSINOMEDI column. The query highlights schools with no Ensino Médio.

IS NOT NULL

IS NOT NULL comparison operator executes the inverse operation of IS NULL operator: select lines with values in the column. Use the popup menu of Favelas theme and select Attribute Query option. Define the attribute as POPULAÇAO and operator as IS NOT NULL. It is also not necessary to type in an entry. Click New Query button.

The query will show Vilas/ Favelas, Conjuntos Habitacionais, Bairros, and Aglomerados with values for population. The same query can be made as NOT POPULAÇAO IS NULL. By adding NOT at the beginning of a command, the selection is inverted.
