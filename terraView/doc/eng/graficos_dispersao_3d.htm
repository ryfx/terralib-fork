Plotting Dispersion 3D Graphics

The interface is activated through the grid area. At least three variables must be selected in the grid to enable "Dispersion 3D" option in the context menu of the grid viewport. The context menu is shown in Figure 1.


Fig. 1 - Context menu to activate the dispersion 3D graphic.

Selected columns are assigned x, y, and z axis, respectively. To change the assignments, swapping of the grid columns must be executed before activating the 3D graphic plot interface.

The dispersion 3D graphic interface is shown in Figure 2. TerraView uses QT and opengl libraries to plot the graphic.


Fig. 2 - Interface to plot a dispersion 3D graphic.

The interface has four drawing areas. The main area displays the data plotted in three dimensions. The remaining drawing areas display the same data plotted in bi-dimensional planes.

The operation is similar to dispersion 2D graphic. Data representing non pointed objects are drawn using theme default color. Data representing pointed objects are drawn using theme pointing color.

"Viewing Control" box contains commands to change the visualization mode of 3D data. Displaying tri-dimensional data in a bi-dimensional area requires controls to visualize data at different angles and positions.

"Show Axis" controls the display of x, y, and z axes. Figure 3 shows the interface without axes display.


Fig. 3 - X, y, and z axes removed.

"Quality" slider controls the quality of 3D points plot. Since each point is represented by a sphere, this slider controls the quality of a sphere drawing. The worst the quality is, the faster the plot is.

"Zoom" slider controls the zoom. Figure 4 shows data display with zoom.


Fig. 4 - 3D data zoom.

Zoom can also be executed through the mouse the middle button or the roller. Drag the mouse upwards to decrease zoom. Drag the mouse downwards to increase zoom.

The illumination considers the light sources: front or back. Figure 5 is similar to Figure 4, but the illumination is from behind.


Fig. 5 - Illumination from behind.

"Rotation" box contains X, Y, and Z sliders to control rotations in x, y, and z, respectively. Figure 6 shows 3D data plot with rotation.


Fig. 6 - 3D data plot with rotation.

Rotation can also be executed by mouse. Press alt key and left button to control rotation in x and y.Press control key and left button to control rotation in z. If LINUX is used, Alt key must be used with Control key simultaneously.

"Selection Control" box controls the origin and the size of selection hexagon. Selection hexagon is displayed with magenta color.

"Offset" and "Size" boxes contain sliders to control the origin and the size of the hexagon in x, y, and z. Figure 7 shows the hexagon in a new location.


Fig. 7 - Selection hexagon in a new location.

Selection control can also be executed by mouse using the three bi-dimensional plane drawing areas. Place the mouse cursor at the center of the rectangle of each drawing area and drag the rectangle to its new location. Place the cursor near the edges to control the rectangle size (hexagon). Figure 8 shows the rectangle being dragged and Figure 9 shows the rectangle being resized.


Fig. 8 - Selection rectangle being dragged (relocated).


Fig. 9 - Selection rectangle being resized.

Selection is executed every time the hexagon changes either size or position. Figure 10 shows the hexagon located on a 3D points cloud. Figure 11 shows the same data in zoom.

Every time the selection rectangle is changed, a query is executed to search for all objects inside the hexagon. However, when the hexagon is changed by mouse, there is an option to only relocate without a query. Use Control key to use this option. To add the new selection to the current one, use Shift key.


Fig. 10 - Hexagon located on a 3D points cloud.


Fig. 11 - Hexagon with zoom located on a 3D points cloud.

Almost all commands by mouse are available in the interface and only the 3D data translation command is executed exclusively by mouse. Translation is executed by dragging the mouse on the 3D drawing area with the left button pressed down.

The module allows also the analysis of a subset of data, using:

  • all data,
  • queried only,
  • non queried only,
  • grouped only, and
  • non grouped only.

Figure 12 shows the list of input data.


Fig. 12 - List with input data options.

In addition, the module allows the analysis of data with 1 to n cardinality. When grouping is not used, all grid data are plotted in the interface. When data is grouped, they are summarized by:

  • MIN,
  • MAX,
  • AVG,
  • SUM, and
  • COUNT.
Figure 13 shows the list with summary option (grouping functions).


Fig. 13 - List with grouping functions options.

Observations on 1 to n cardinality.

The module considers that an object is pointed if at least one of its attributes lines is pointed. Therefore, its graphic representation is drawn on the map using the theme pointing color. Other modules, such as histogram, dispersion, and dispersion 3D, draw objects according to:

- When there is no grouping function, all table values are used to plot the graphic. In this case, only points or bars related to the attributes pointed on the grid are drawn using the theme pointing color.

- When a grouping function is used, the summarized values for each object are used to plot the graphic. Therefore, by pointing to a line of its attributes, an object is considered as pointed. Similarly, by satisfying a line of its attributes in a query, an object is considered as queried.

- These observations are important when the input data is filtered by queried, not queried, grouped, and not grouped.
