Importing Attribute Tables

 

When a table with additional information (attributes) about geographic data of a layer exists, this information can be included in this layer using TerraView. The resulting table will be another Attribute Table of that layer.

Tables of this type must be in one of these formats: DBF (.dbf); Ascii/Text (.csv or .txt); Spring (.spr) and Access (.mdb). When in the DBF, Spring or Access formats, column names and types are already stored in the file that contains the table, whereas in the Text format they must be specified by the user when importing the file. In this example, the table is in the DBF format. If it were in Text format, you should follow the instructions on providing specific information about table columns, that are in the preceding section, Importing Point Data.

To import this type of table, use the File > Import Table menu option . In the Import Table interface, fill in the required parameters.

 

  1. Click on File button and select the file with the atribute table to be imported. In this example, select the file SOC_ECONOMY.dbf, which is available in TerraView tutorial. Don´t forget to change the file selection filter to DBF files (*.dbf) in order to see only files with this extension.
  2. Select Static as the Type of Destination Table, which means an attribute table.
  3. Select the Layer to which the new attribute table will be associated, in this case Districts1.
  4. Select the Geometry link, the attribute of the table used to link to the geometries. In this example, use DIST.
  5. Fill in the Name of the table, in this example, leave the suggested name: SOC_ECONOMY.
  6. Apply the import by clicking OK button.
    7.


Now the Districts layer has two attribute tables: one that is part of the original mid/mif file and the other, SOC_ECONOMY, imported from the DBF file. The themes associated with this layer may have one or both of these tables visibles on the grid.

In order to visualize SOC_ECONOMY table data in the grid, the table has to be included in the selected Theme. In Views/Themes Tree, follow the steps below:

  1. Click Districts theme to activate it.
  2. Activate the theme popup menu (right click on its name).
  3. Click on Select Theme Tables option.


In the Selection of Theme Tables interface you shall follow the steps e notes below.

  1. The left panel shows the tables of the layer that are not currently associated with the theme;
  2. The right panel shows the tables that are currently associated with the theme;
  3. The arrows allow you to transfer a table from one side to the other;
  4. Transfer SOC_ECONOMY table to the right side (Theme Tables) and click on OK.


As you can see below, the attributes of the table that was imported are now displayed in the grid, e.g. RENDA_FAM which is displayed in the status bar as belonging to SOC_ECONOMY table.
