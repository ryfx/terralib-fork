Bayesian Empirical Ratios

In this section, ratios from attributes of map objects will be created. The first step calculates the gross ratio of homicides in the year 2000 for Belo Horizonte neighborhoods. Next, use Global Bayes Empirical tool to adjust the ratios.

Gross ratio is the ratio between a numerator (number of homicides, kidnappings, deaths) and a denominator (usually population at risk). Gross ratio presents high instability for regions with small population at risk, that is, regions with small population are more susceptible to variations in the ratios due to few random events than regions with larger population. Global Bayesian empirical ratio is based on the use of information from other regions of the study area to reduce the instability not associated to the occurrence of risk event.

The first TerraView requirement to calculate Bayesian ratios is that the denominator (population at risk) does not have values equal or smaller than zero. If there are missing population data for some Belo Horizonte neighborhoods, a layer with neighborhoods with data must be created.

Gross ratio calculation

To calculate gross ratio, create a new column in neighborhoods table (Bairros Theme) and update this column data. To create a new column, use the popup menu from a column header in the grid area and select Add Column option.

After creating a column named Ratio, use this column popup menu from and change its data:

Apply the operation where the numerator is the number of homicides in the year 2000 and the denominator is the population of BH neighborhoods in the same year. Multiply it by 100,000 to standardize the values.

Bayesian empirical ratio calculation

Next, create a column with global Bayesian empirical ratio to adjust the ratios for municipalities with a small population. To create it, use Analysis menu and Global Empirical Bayes. An interface to set the parameters will be displayed as below

Set the following parameters:

1. Select Bairros theme in Theme field.

2. In Population in Risk field, select the table and the column with population data: Bairros and POP2000.

3. In Number of Cases field, select the table and the column with the ratio numerator data (homicides in 2000): Bairros and HOM2000.

4. Select 100000 in Multiplicative Rate Correction field.

5. Apply to create the new column named GEBayes. If required, change the name in New Column Name field.
