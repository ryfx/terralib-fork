Plugin Cópia de Banco de Dados

O Plugin Cópia de Banco de Dados cria um banco de dados com a cópia do banco de dados ativo no TerraView. Isso permite que um banco de dados seja convertido para um outro tipo de banco mantendo todas as suas informações, como planos de informação, vistas, temas, tabelas estáticas e externas. Antes de acessar esse plugin, deve-se então abrir o banco de origem que possui os dados a serem copiados.

A figura abaixo mostra a primeira janela do Plugin de Cópia de Banco de Dados. Nessa janela é habilitada somente a opção Criar, pois o plugin criará um banco e depois converterá os dados do banco ativo para o banco criado. Para o banco Oracle e SQL Server é necessário que o usuário crie previamente um banco de dados vazio, uma vez que esses tipos de bancos requerem que a criação seja feita pelo DBA ou administrador do banco.

Siga os o procedimento abaixo para o preenchimento dos Parâmetros de Conexão com o banco de dados:

1. Selecione o SGDB na lista Tipo do Banco. Observe que a interface se altera conforme o SGDB escolhido:

a) No caso do Access, clique no botão Diretório para localizar onde o novo banco de dados será armazenado.

b) Para o caso dos outros SGBD's que possuem uma arquitetura cliente/servidor, é necessário informar a localização do servidor (campos Servidor ou Nome do Serviço e Porta) e quais os parâmetros de acesso (campos Usuário e Senha). Ainda nesse caso, clicando no botão Listar podemos informar a Codificação desejada para o banco de dados (UTF8 ou LATIN1 por exemplo).

2. No campo Nome do Banco informe o nome do banco de dados a ser criado.

3. Clique no botão Próximo.

Criar um banco de dados significa criar tabelas e relacionamentos que serão futuramente usados pelo TerraView. Após criar um banco ele fica automaticamente conectado ou aberto e pronto para o próximo passo da cópia de dados.
