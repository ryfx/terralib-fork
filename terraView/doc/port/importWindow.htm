Importação de Dados Vetoriais

O TerraView é capaz de importar dados vetoriais nos seguintes formatos: Shapefiles, MID/MIF, TAB/GEO vindo do SPRING e BNA vindos do TabWIN. Para executar a importação de um dado vetorial um banco de dados deve ter sido previamente conectado.

MID/MIF

Suponha que os arquivos distritos.mif e distritos.mid, contém um mapa dos distritos do município de São Paulo com suas localizações e seus atributos descritivos. As extensões ".mif" e ".mid" indicam que esses dados estão no formato de intercâmbio do software MAPInfo, que pode ser importado pelo TerraView. Para importar esse dado execute os seguintes passos:

  3. Veja na figura abaixo que a interface de importação reconhece a projeção do dado, pois esse formato possui essa informação. A interface também sugere um nome para o Plano de Informação, ou camada, que irá conter seu dado.

Seguindo a interface, a caixa Ligação entre Tabelas de Atributos e Geometrias pede que você informe como será a ligação entre os atributos dos distritos (no arquivo MID) e as suas geometrias (no arquivo MIF). A opção Automática indica que isso será criada um atributo a mais com a numeração sequencial dentro do arquivo e esse atributo será usado para ligar os dois arquivos. A opção Selecionar Coluna permite que você use um dos atributos (ou colunas) já existentes no arquivo de atributos. Se você usar essa opção certifique-se de escolher um atributo, ou coluna, que seja não se repita para os objetos do arquivo. Shapefile

A importação de dados em formato shapefile é semelhante ao visto na seção anterior. Suponha agora que se deseje importar um arquivo de dados que contém pontos com localizações das estações de metrô, dessa vez em formato Shapefile, formato de intercâmbio dos produtos ESRI. Shapefiles são formados por três arquivos: "estacoes-metro.shp", "estacoes-metro.dbf" "estacoes-metro.shx". Siga os passos 1 a 3 descritos acima. Lembre-se de alterar o filtro no passo 2. Você vai obter a interface como mostrada na figura abaixo:

 


Observe dessa vez, que a projeção não foi reconhecida do formato e a interface indica que o dado está sem projeção "NoProjection". Se você conhece a projeção do seu dado, clique no botão Projeção... e informe através da interface de definição de projeção mostrada na figura abaixo.

 


Para cada projeção um conjunto de parâmetros deve ser fornecido. Nesse caso dos dados sobre as estações do metrô estão na projeção UTM, Zona 23, com Datum SAD69. Informe esses parâmetros e clique no botão OK. Termine a importação conforme o exemplo anterior (passos 4, 5 e 6). A coluna ROTULO pode ser usada como coluna de ligação. Dessa vez, responda "Não" a pergunta sobre se deseja visualizar os dados importados. Observe então que o plano ficará disponível (veja a figura abaixo) mas não é automaticamente desenhado.

 

Tab/Geo

Vamos importar agora um arquivo de dados que contém pontos com localizações das estações de ferroviárias, em formato Geo/Tab, formato de exportação de dados do SPRING para o TerraView. Esse formato é definido por 2 arquivos: "estacoes-ferroviarias.geo" e "estacoes-ferroviarias.tab. Siga os passos 1 a 3 descritos acima. Lembre-se de alterar o filtro no passo 2. Esse formato também não traz informação sobre a projeção do dado. Se conhecê-la informe a projeção do dado como mostrado para o  formato Shapefile acima.  Observe que para esse formato, não é possível escolher qual a coluna de ligação uma vez que essa já vem determinada pelo dado no SPRING. Portanto, o passo 4 não é válido para esse formato. Novamente, se a resposta para a visualização automática for "Não" o plano ficará disponível como mostrado na figura abaixo, mas o dado não será automaticamente desenhado.
