Importando Tabelas de Atributos

 

Se você possui uma tabela com dados adicionais sobre os objetos geográficos de um plano de informação, é possível incorporá-la ao banco de dados na forma de mais uma tabela de atributos daquele plano.

As tabelas de atributos podem estar nos formatos: DBF (.dbf), Ascii/Text (.csv ou .txt), Spring (.spr) ou Access (.mdb). Nos formatos DBF , Spring e Access os nomes e tipos das colunas estão definidos no próprio arquivo que contém a tabela, enquanto no formato texto eles devem ser informados durante a importação. No exemplo abaixo, a tabela de atributos está no formato DBF. Se ela estivesse no formato texto, você deveria seguir as instruções da secção precedente, Importando Tabelas de Pontos, sobre a especificação de nomes e tipos das colunas da tabela.

Para importar uma tabela como esta, você deve utilizar a opção de menu Arquivo > Importar Tabela. A janela Importar Tabela será aberta para que você preencha os parâmetros necessários.

 

  1. Pressione o botão Arquivo e selecione o arquivo SOC_ECONOMY.dbf, disponível no tutorial do TerraView. O campo Encoding, bem à direita nome do arquivo, quando estiver habilitado, permite que se possa especificar uma codificação para os dados do arquivo de importação diferente da utilizada internamente no banco de dados. Normalmente pode ser deixado em branco
  2. Informe o tipo de tabela. Selecione Estático (significando que se trata de uma tabela de atributos).
  3. Selecione a qual Plano de informação a tabela de atributos estática está associada. Nesse caso ao plano Districts1.
  4. Selecione qual coluna/atributo deve ser usado para fazer a ligação com as geometrias dos distritos. Nesse caso a coluna DIST.
  5. Selecione o nome para essa tabela. Observe que o sistema lhe oferece uma opção default.
  6. Clique no botão Executar.
    7.


Agora o plano de informação possui duas tabelas de atributos alfanuméricos ou descritivos: uma é parte do arquivo original shapefile e a segunda importada do arquivo DBF. Os temas associados com este plano podem ter uma ou duas destas tabelas visíveis na janela de grade.

Para visualizar os atributos da tabela SOC_ECONOMY na janela de grade, esta tabela deve ser incluída no tema selecionado. Na árvore de vistas e temas, siga os passos abaixo:

  1. Clique no tema Districts1 para torná-lo o tema ativo.
  2. Acione o menu popup do tema (botão direito sobre seu nome.
  3. Clique na opção Selecionar Tabelas do Tema.


A janela Seleção das Tabelas do Tema será mostrada, conforme abaixo. Observe os comentários e instruções a seguir.

  1. O lado esquerdo da tela mostra as tabelas que pertencem ao plano de informação e que não estão sendo usadas pelo Tema;
  2. O lado direito mostra as tabelas que o tema está correntemente usando;
  3. As setas permitem que você transfira uma tabela de um lado para o outro;
  4. Transfira a tabela SOC_ECONOMY para o lado direito (Tabelas do Tema) e clique em Executar.


Você pode notar que os atributos da tabela que foi importada agora aparecem na janela de grade. Este é o caso do atributo RENDA_FAM que, conforme mostrado na barra de status, está localizado na tabela SOC_ECONOMY.
