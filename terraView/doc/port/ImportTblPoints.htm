Importando Dados de Pontos

 

Você pode possuir informações geográficas sobre pontos, na forma de tabela. A posição geográfica de cada ponto, definida pelas coordenadas X e Y, pode ter sido, por exemplo, obtida através do uso de um GPS . É necessário que cada linha da tabela corresponda a um único ponto.

As tabelas de dados de pontos podem estar nos formatos DBF ou texto. No formato DBF os nomes e tipos das colunas estão definidos no próprio arquivo que contém a tabela, enquanto no formato texto eles devem ser informados durante a importação. No exemplo abaixo, a tabela de pontos se encontra no formato texto, também dito CSV, que é uma sigla para "Valores Separados por Vírgula" em inglês.

Para importar uma tabela como esta, você deve utilizar a opção de menu Arquivo > Importar Tabela de Pontos. A janela Importar Tabela de Pontos será aberta para que você preencha os parâmetros necessários.

Siga os seguintes passos:

  1. Selecione o arquivo com a tabela de pontos. Os formatos disponíveis são DBF ou CSV. Pressione o botão Arquivo e selecione o arquivo fire_spot_sp.csv, disponível no tutorial do TerraView (lembre-se de alterar o filtro de seleção de arquivos para buscar aqueles com extensão “.csv”). O campo Encoding, ao lado do nome do arquivo, quando estiver habilitado, permite que se possa especificar uma codificação para os dados do arquivo de importação diferente da utilizada internamente no banco de dados. Normalmente pode ser deixado em branco.

  2. Responda à questão: “Entre com o caractere separador (dos valores das colunas em cada linha)“ escolhendo um dos caracteres disponíveis na interface, ou algum outro de sua escolha. Nesse caso o caractere é Vírgula.

  3. Informe se a primeira linha dentro do arquivo é um cabeçalho com o nome das colunas. Nesse caso a resposta é Sim.

  4. Para cada coluna presente no arquivo, devem ser informados: seu tipo, nome, número máximo de caracteres (se for do tipo texto) e o formato usado (se for do tipo data/hora). Para as alterações de cada uma das colunas serem efetivadas, clique no botão Aplicar.

  5. Escolha o nome do plano a ser criado.

  6. Informe, se souber, a projeção na qual as coordenadas dos pontos estão descritas. Nesse caso elas estão na projeção LatLong, Datum WGS84.

  7. Indique na área Informação da Geometria quais colunas correspondem às coordenadas dos pontos. Nesse caso, a coluna Lon contém a coordenada X e a coluna Latitude contém a coordenada Y. Elas foram definidas como do tipo Real, conforme destacado na figura, porque estão em graus decimas. Se elas estivessem em graus, minutos e segundos, como as colunas LatGMS e LonGMS mostradas na figura, elas deveriam ser definidas na interface como do tipo Texto, sendo importadas corretamente e podendo ser usadas como coordenadas dos pontos. Os valores numéricos nestas colunas devem usar sempre como separador decimal entre a parte inteira e a parte fracionária o caractere "." (ponto).

  8. Informe se alguma das colunas serve como identificador único de cada objeto representado pelo ponto. Ela será usada como ligação entre a tabela de atributos e os objetos (geometrias). Se não houver uma coluna nesta situação, use a opção Automático. Nesse exemplo, use a coluna Nr.

  9. Clique no botão Executar.
